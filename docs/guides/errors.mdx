## Overview

The Verbitto program defines **35** custom error codes under the `VerbittoError` enum. Anchor assigns each variant a numeric code starting at 6000.

## Error Code Table

| Code | Name | Message |

| 6000 | `InvalidFee` | Fee basis points must be ≤ 5000 (50%) |
| 6001 | `InvalidConfig` | Invalid platform configuration |
| 6002 | `BountyTooLow` | Bounty is below the platform minimum |
| 6003 | `TitleTooLong` | Title exceeds 64 characters |
| 6004 | `DeadlineInPast` | Deadline must be in the future |
| 6005 | `InvalidRepReward` | Reputation reward must be 0–1000 |
| 6006 | `TaskNotOpen` | Task is not in Open status |
| 6007 | `TaskExpired` | Task has passed its deadline |
| 6008 | `TaskNotClaimedOrRejected` | Task is not in Claimed or Rejected status |
| 6009 | `NotAssignedAgent` | Caller is not the assigned agent |
| 6010 | `TaskNotSubmitted` | Task is not in Submitted status |
| 6011 | `NotTaskCreator` | Caller is not the task creator |
| 6012 | `DeadlineNotReached` | Deadline has not been reached yet |
| 6013 | `TaskCannotExpire` | Task cannot be expired in its current status |
| 6014 | `TemplateInactive` | Template is not active |
| 6015 | `TaskNotDisputable` | Task is not in a disputable status |
| 6016 | `NotTaskParty` | Caller is not a party to this task |

## Client-Side Handling

Anchor wraps program errors in an `AnchorError`. Parse them on the client to display user-friendly messages.

```typescript
import { AnchorError } from '@coral-xyz/anchor';

try {
  await program.methods.claimTask().accounts({ ... }).rpc();
} catch (err) {
  if (err instanceof AnchorError) {
    switch (err.error.errorCode.code) {
      case 'TaskNotOpen':
        toast.error('This task is no longer available.');
        break;
      case 'PlatformPaused':
        toast.error('Platform is paused. Try again later.');
        break;
      default:
        toast.error(err.error.errorMessage);
    }
  } else {
    // Wallet rejection, network error, etc.
    toast.error('Transaction failed. Please try again.');
  }
}
```

<Callout type="info">
The `errorCode.code` field is the string variant name (e.g., "TaskNotOpen"), while `errorCode.number` is the numeric code (e.g., 6006).
</Callout>
